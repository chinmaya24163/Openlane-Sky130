# OpenLane Flow Architecture

The OpenLane flow acts as a wrapper integrating multiple tools together.

## Some Open-Source Tools Used

| Flow Stage | Tool |
|---|---|
| RTL Synthesis | Yosys |
| Logic Optimization | ABC |
| Static Timing Analysis | OpenSTA |
| Physical Design | OpenROAD |

## OpenLane Input and Output

### Inputs

- RTL Verilog source
- PDK files

### Output

The final output is GDSII Layout used for fabrication.

---

# Interactive Flow Execution

Commands must be executed in correct order.

## Correct Flow Sequence

```text
prep
 ↓
run_synthesis
 ↓
run_floorplan
 ↓
run_placement
 ↓
run_cts
 ↓
run_routing
 ↓
signoff
```

## Why Flow Order Matters

Each stage depends on outputs generated by previous stages. Example:

- routing requires CTS outputs
- CTS requires placement outputs

If a step is skipped subsequent stages fail.

# Fully Automated Flow

OpenLane also supports Fully Automated RTL-to-GDSII Execution without step-by-step interaction.

## Automated Flow Command

```bash
./flow.tcl -design <design_name>
```

This executes the complete ASIC flow automatically.
